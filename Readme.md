# 弹幕源计划（Danmaku Source Project）

## 📌 项目简介

“弹幕源计划”是一个以 **弹幕数据聚合与分发为核心** 的跨平台系统，优先支持 Android 端，并逐步扩展至 Web 与 Windows。

与传统弹幕外挂项目不同，本项目的核心目标是：

> **将第三方平台弹幕（如 B站）抓取、清洗、存储到自有服务器，构建统一弹幕资源库，并对外提供服务。**

---

## 🎯 核心定位

* ❗ 不只是“显示弹幕”
* ✅ 而是构建“弹幕数据基础设施”

---

## 🧱 总体架构

```
         ┌────────────────────┐

```

## ⚙️ 技术选型

### 🦀 后端（核心）

* Rust

#### 选型原因：

* 高并发（适合弹幕流）
* 内存安全
* 低延迟

#### 技术组件：

* 网络框架：axum / actix-web
* 异步运行时：tokio
* 数据解析：quick-xml
* 数据库：PostgreSQL / SQLite（初期）
* 缓存：Redis（弹幕流）

---

### 📱 前端（优先 Android）

* Flutter

#### 选型原因：

* 一套代码多端复用
* 高性能动画（适合弹幕）

---

## 🔌 核心模块设计

### 1️⃣ 弹幕采集模块（Crawler）

功能：

* 抓取第三方弹幕（B站 XML）
* 支持视频ID解析

特点：

* 可扩展（未来支持多平台）

---

### 2️⃣ 弹幕解析与标准化（Parser）

统一弹幕数据结构：

```json
{
  "time": 12.5,
  "content": "hello",
  "type": "scroll",
  "color": "#ffffff",
  "source": "bilibili"
}
```

处理内容：

* XML → JSON
* 时间排序
* 格式统一

---

### 3️⃣ 弹幕清洗与去重

目的：

* 避免重复弹幕
* 提高质量

策略：

* 内容哈希去重
* 时间窗口过滤

---

### 4️⃣ 弹幕资源库（核心差异点）

这是本项目与传统弹幕插件的本质区别：

> ❗ 所有弹幕将存入“自有服务器”形成数据资产

功能：

* 按视频ID索引
* 支持多来源融合
* 支持快速查询

---

### 5️⃣ 弹幕分发服务

协议：

* WebSocket（实时）
* HTTP（历史）

功能：

* 按时间轴推送
* 支持多客户端同步

---

### 6️⃣ 客户端渲染（Flutter）

模块划分：

```
DanmakuEngine
├── Scheduler（时间调度）
├── TrackManager（轨道管理）
└── Renderer（绘制）
```

---

## 📊 与开源项目差异

| 维度    | 传统弹幕插件 | 弹幕源计划 |
| ----- | ------ | ----- |
| 数据来源  | 实时获取   | 本地资源库 |
| 数据存储  | 无      | 有（核心） |
| 多平台融合 | 弱      | 强     |
| 可扩展性  | 低      | 高     |
| 价值    | 工具     | 平台    |

## 🚀 开发阶段规划（六阶段）

本项目采用 **6阶段渐进式开发**：

* 前三阶段：后端（数据能力）
* 后三阶段：前端（体验能力）

设计理念：

> **功能逐步可用 + 每阶段可独立运行 + 持续积累数据资产**

---

### 🦀 第一阶段：基础弹幕采集（后端 MVP）

目标：打通最小数据链路

* [ ] B站视频ID解析
* [ ] 弹幕XML抓取
* [ ] 基础XML解析
* [ ] 命令行输出弹幕

阶段成果：

* 能获取并解析一条完整弹幕流

---

### 🦀 第二阶段：数据标准化与存储（后端核心）

目标：建立“弹幕资源库”雏形

* [ ] 统一弹幕数据结构（JSON）
* [ ] 数据库存储（PostgreSQL / SQLite）
* [ ] 基础去重（hash + 时间窗口）
* [ ] 视频ID索引体系

阶段成果：

* 弹幕可持久化查询

---

### 🦀 第三阶段：弹幕服务化（后端完成）

目标：提供对外服务能力

* [ ] HTTP API（获取历史弹幕）
* [ ] WebSocket 实时推送
* [ ] 弹幕时间排序优化
* [ ] 简单缓存（Redis）

阶段成果：

* 后端成为“弹幕服务器”

---

### 📱 第四阶段：Android 基础客户端（前端 MVP）

目标：能看到弹幕

* [ ] Flutter 项目初始化
* [ ] 接入后端 API
* [ ] 基础弹幕渲染（无轨道优化）
* [ ] 视频时间手动同步

阶段成果：

* 安卓端可显示弹幕

---

### 🎨 第五阶段：弹幕引擎与视觉优化（前端核心）

目标：达到“可用体验”

* [ ] 轨道算法（防重叠）
* [ ] 平滑滚动动画
* [ ] 弹幕样式系统（字体 / 透明度 / 描边）
* [ ] 性能优化（高密度弹幕）

🎯 设计风格（Claude美学）：

* 极简（Minimalism）
* 高留白（Whitespace）
* 低干扰（Non-intrusive）
* 柔和动画（Smooth & Calm）

阶段成果：

* 接近专业弹幕体验

---

### 🌐 第六阶段：多端扩展与产品化

目标：形成完整产品

* [ ] Web 端支持（Flutter Web）
* [ ] Windows Overlay
* [ ] 多平台弹幕融合展示
* [ ] 用户配置系统

🎯 Claude风格深化：

* 动效自然（无突兀变化）
* 信息层级清晰
* UI不抢内容（弹幕不喧宾夺主）

阶段成果：

* 完整跨平台弹幕系统

---

## 🧠 核心难点

### 🚧 1. 数据规模增长

* 弹幕量巨大

解决：

* 分库分表
* 缓存层（Redis）

---

### 🚧 2. 多源弹幕融合

* 不同平台格式不同

解决：

* 统一标准结构
* Source 标记

---

### 🚧 3. 时间轴对齐

解决：

* 客户端校准
* 偏移参数
